# Support

Thanks for using **AutoTriage**. This page explains where to get help.

## Before you ask

- **Running it / expected output:** [docs/runbooks/running.md](docs/runbooks/running.md)
- **An error or surprising result:** [docs/runbooks/troubleshooting.md](docs/runbooks/troubleshooting.md)
  (symptom → cause → fix table)
- **Configuration (env vars & flags):** [docs/configuration.md](docs/configuration.md)
- **How it works:** [README.md](README.md) and [docs/architecture.md](docs/architecture.md)
- **Contributing:** [CONTRIBUTING.md](CONTRIBUTING.md)

Many issues (missing scanner binary, no API key, empty findings) are covered in
the troubleshooting runbook.

## Getting help

- **Questions & ideas:** open a [GitHub Discussion](https://github.com/rohanbatrain/autotriage/discussions)
  (or a Q&A issue if Discussions are not enabled).
- **Bugs:** open an issue with the
  [Bug report template](https://github.com/rohanbatrain/autotriage/issues/new?template=bug_report.md).
  Include the exact command, the run summary output, any warnings, and your
  Python/scanner versions.
- **Feature requests:** open an issue with the
  [Feature request template](https://github.com/rohanbatrain/autotriage/issues/new?template=feature_request.md).

## Reporting a security vulnerability

**Do not** open a public issue for a security vulnerability in AutoTriage itself.
Report it privately via GitHub's
[private security advisories](https://github.com/rohanbatrain/autotriage/security/advisories/new),
or contact the maintainer [@rohanbatrain](https://github.com/rohanbatrain)
directly. We will acknowledge and work with you on a fix and coordinated
disclosure.

## Response expectations

AutoTriage is an open-source project maintained on a best-effort basis. There is
no SLA on community support. Well-described, reproducible reports get triaged
fastest — which is rather on-brand.
