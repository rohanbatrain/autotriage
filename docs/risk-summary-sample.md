## Top risks by business impact

1. **Insecure deserialization / RCE (`pickle.loads` on request data) — Critical.** An attacker can achieve remote code execution on the service host by sending a crafted payload. Full host and lateral-movement risk. *(CWE-502)*
2. **`eval()` on request data — Critical.** Direct arbitrary-code execution from an HTTP parameter. Trivially exploitable. *(CWE-95)*
3. **SQL injection on user lookup — Critical.** Unauthenticated injection could expose or modify the entire customer/payments datastore. *(CWE-89)*
4. **Vulnerable PyYAML 5.1 (CVE-2020-1747) — Critical.** Arbitrary code execution via `FullLoader`; a single dependency bump closes it. *(CWE-502)*
5. **OS command injection (`os.system`) — Critical.** Untrusted host value concatenated into a shell command; enables command execution. *(CWE-78)*
6. **Hardcoded AWS access key in source — High.** A live-looking credential is committed; assume compromised, rotate now, and purge from history. *(CWE-798)*
7. **Internet-exposed infrastructure — High.** A public-read S3 bucket (any object world-readable) and a security group allowing SSH from the entire internet materially expand the attack surface. *(CWE-732, CWE-284)*
